DnD World & Campaign Manager — Design Overview
Core Goals

A flexible, lightweight worldbuilding and campaign management tool for D&D-style TTRPGs.
The system should allow users to create interconnected data entries (e.g. cities, people, factions, myths) that can be freely linked and organized into a custom hierarchy.

Key Features
1. Dynamic Hierarchy

No fixed structure (e.g. no forced “World > Country > City”).

Users can freely create and nest folders or “collections” to represent any kind of grouping.

Each entry can contain any number of sub-entries (e.g. People, Families, Myths).

Hierarchies are entirely user-defined and editable at any time.

2. Object Model

Each object (or “entry”) contains:

id: a UUID-style identifier, e.g. 03a2a911_beowulf
(prefix from category/type + unique identifier)

name: display name, e.g. “Beowulf”

type: optional category, e.g. “Person”, “City”, “Faction”

parent_id: optional link to a parent object

children: array/list of sub-entries

description: rich text (Markdown or similar), supports inline linking

images: list of image references (with one marked as “default”)

tags: optional metadata tags

created, updated, deleted timestamps (soft delete model)

3. Linking System

Any word or phrase in a description can be linked to one or more objects.

Selecting a word → Right-click → “Link” → Choose existing entries.

Hovering over a linked word shows a popup with:

Name

Short description

Default image (if one exists)

If multiple entries share the same name (e.g. two “Ragnar”s), clicking shows a dropdown of linked items.

4. Editing & Navigation

Fast, lightweight editor UI with keyboard shortcuts.

Simple Markdown or WYSIWYG editor with inline linking.

Sidebar for navigating hierarchy.

Search bar for finding any entry quickly (by name, tag, or id).

Drag-and-drop to reorganize hierarchy.

5. Attachments

Each object can have one default image plus additional images.

Optional attachments (maps, documents, etc.) stored locally or via file references.

6. Data Structure Example
{
  "id": "03a2a911_beowulf",
  "name": "Beowulf",

  "deleted": null
}

7. Linking Example

In the editor:
The city of [[Heorot]] is ruled by [[Hrothgar]].

“[[Heorot]]” and “[[Hrothgar]]” automatically link to entries with matching names or show a menu if multiple exist.

8. UI Concept

Left panel: hierarchy browser (expandable tree view)

Main panel: entry view/editor (name, type, description, images)

Right panel (optional): links, tags, metadata

Hover popup: short info + image for any linked item

9. Backend / Storage Options

Local JSON database (for simplicity and portability)

Each object can be stored as a separate JSON file or as part of a master JSON index

Future option: SQLite for performance if scaling up
